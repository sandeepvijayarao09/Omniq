Privacy-First Architecture

Zero raw PII reaches any model — ever.

Every AI request routes through Omniq before touching any model. The privacy engine scans every input for 14 categories of sensitive data — names, emails, financial identifiers, health information, IP addresses, classified project names — and replaces each with a synthetic token. The model reasons over tokens. The real data never leaves the building.

  • 14-category automatic PII tokenization at the edge
  • Architecture-enforced — not a policy, a technical guarantee
  • One-command GDPR erasure for any user on request
  • HIPAA, SOC 2, and ISO 27001 compliance posture maintained

Architecture

How Omniq works

A layered intelligence architecture that puts security and org context at every step — not bolted on at the end.

01

Employee sends a request

Any employee, any device. Natural language. No prompting skills required. Omniq understands intent in the context of their role.

↓
02

Auth layer validates identity & permissions

JWT validation and Role-Based Access Control. The employee's role determines exactly which data categories and documents they can access — enforced architecturally, not by policy alone.

03

RAG engine retrieves relevant org context

Access-filtered vector search across your indexed documents. Only content the employee is authorized to see is retrieved. Semantic search, not keyword matching.

↓
04

PII tokenization before any model call

14-category tokenizer strips all sensitive data. Customer names become tokens. Financial figures are anonymized. The model context is clean — organizationally rich but personally anonymized.

05

LLM generates a grounded, accurate response

Claude processes org context + retrieved knowledge and generates an answer grounded in your actual documents, values, and current priorities.

↓
06

Response delivered + interaction logged

The employee gets their answer. Every detail of the interaction — query, documents accessed, model used, response — is written to the immutable audit log.
